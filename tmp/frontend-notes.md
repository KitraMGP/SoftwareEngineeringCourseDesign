## 已确认事实

- 项目 V1 目标是“私有知识库问答系统”
- 后端主设计文档仍以 `docs/detailed-design.md` 为准
- 当前仓库还没有正式的前端工程代码
- 当前原型图只有登录页和聊天页
- 前端范围本轮按“用户端 + 管理端一起设计”推进
- 前端应用形态按“单仓库双应用”推进
- 已确认的工程基线：
  - `Vue 3 + TypeScript + Vite`
  - `Pinia + Vue Query`
  - `TailwindCSS + SCSS`
  - 基础组件库采用 `Element Plus`
  - `pnpm`
  - `ESLint + Prettier + Vitest + Playwright`
- 已确认的产品与交互基线：
  - 聊天页左侧采用“顶部会话列表 + 中部一级导航 + 底部用户入口”
  - 知识库入口位于聊天页侧边抽屉
  - 注册为独立页面
  - 个人资料与修改密码放在独立个人中心
  - 一级导航当前按“知识库 / 关于”理解
  - 关于页包含：产品介绍、使用帮助、项目/版本信息
  - 关于页为登录后页面
  - 知识库模块采用“列表页 + 详情页”
  - 知识库详情页采用“顶部信息 + 文档列表与上传区”
  - 新建会话时不暴露模型选择
  - 注册页字段包含：用户名、邮箱、密码、确认密码、协议勾选
  - 首期不开放头像编辑
  - 引用来源显示在助手消息下方
  - 未命中知识库时，在助手消息头部展示标签
  - 文档状态在知识库详情页轮询
  - 聊天接口未完成阶段，使用本地假数据演示页面但禁用真实发送
  - 聊天页显示当前绑定知识库状态条
  - 无知识库会话不额外提示
- 已确认的视觉基线：
  - 用户端延续原型的轻柔、空气感、插画气质，但做高级化处理
  - 聊天页偏沉浸式 AI 对话空间
  - 登录页沿用背景图 `frontend/UI原型/登录界面背景图.jpg`
  - 登录背景上覆盖黑色半透明玻璃层以降低亮度
  - 登录表单区采用中央悬浮玻璃卡片
  - 聊天页仅助手消息显示头像
  - 字体采用更有识别度的 Web 字体组合
  - 没有原型的页面在正式文档中补低保真结构说明
- 已确认的管理端信息架构基线：
  - 一级菜单包含：Dashboard、用户、模型配置、任务、审计、系统设置
  - 配额管理放在系统设置下的二级菜单
  - Dashboard 以用户数、知识库数、文档数、任务状态为核心摘要
- 用户端真实可联调的后端能力主要集中在：
  - 认证
  - 当前用户
  - 会话 CRUD
  - 知识库 CRUD
  - 文档上传、列表、详情、删除
  - 重建索引任务创建
